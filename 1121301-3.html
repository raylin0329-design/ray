我的證照

以下為高中期間取得的資訊與文書相關證照， 透過不斷練習與學習， 提升自己的資訊處理、文書設計與程式設計能力。

電腦軟體應用－丙級技術士

學習內容

學習文書處理、資料編排與簡報製作等基礎資訊應用能力。課程內容包含文字格式設定、表格製作、版面配置、資料輸入與檔案管理等實務操作。

遇到的困難

  • 對文書排版與軟體操作不夠熟悉，尤其在術科練習時，必須在有限時間內完成文件編排、格式設定與資料處理，任何細節錯誤都可能被扣分。剛開始練習時，我經常因為格式設定不一致、操作步驟不熟悉，以及對快捷鍵不夠熟練，導致完成速度過慢，甚至無法在時間內完成題目。
  • 此外，由於術科題目重視細節與準確性，只要邊界、字型、間距或表格設定有些微錯誤，就可能影響整體成績，因此我在初期常因小細節失誤而反覆修改，增加許多挫折感。

解決方式

  • 面對這些問題，我開始重新調整自己的練習方式，而不只是反覆操作題目。我會先分析自己容易失分的地方，例如排版順序、格式設定與操作流程，再針對弱點進行重複練習。
  • 同時，我也透過整理操作步驟、熟記快捷鍵，以及計時模擬考試的方式，提升作答效率與穩定度。
  • 當遇到不熟悉的功能時，我會主動查詢教學資料、觀看影片，並透過反覆測試找出正確的操作方式，而不是只依賴課堂上的內容。
  • 透過不斷修正錯誤與調整方法，我逐漸建立起更有系統的問題分析能力與操作邏輯。

學習收穫

透過準備電腦軟體應用丙級證照，我不僅提升了文書處理與電腦操作能力，也學習到「細節管理」與「時間控制」的重要性。這段過程讓我體會到，資訊能力不只是會操作軟體，更重要的是如何在有限時間內，透過邏輯思考與問題分析，完成高品質的成果。

此外，我也培養了自主學習與持續修正的能力。當遇到問題時，我開始學會先分析原因，再尋找適合的解決方式，而不是只依賴他人協助。這樣的經驗也讓我對資訊領域產生更深的興趣，並希望未來能持續提升自己的數位應用與資訊整合能力。

電腦軟體應用－乙級技術士

學習內容

學習到更進階的文書處理與資料整合能力，包含大量文件編排、進階表格設計、資料格式統整、長文件排版以及跨軟體資料應用（如文書與試算表整合）。

遇到的困難

  • 時間壓力高，流程不夠熟練：題目步驟多且環環相扣，會因為前面錯誤而影響後續結果。
  • 細節要求更嚴格：例如格式一致性、段落設定、表格對齊，只要細節錯誤就會影響分數。
  • 操作流程沒有系統化：初期練習時多半是「邊做邊想」，導致效率低且容易遺漏步驟。

解決方式

  • 拆解題目流程：先把題目拆成「資料整理 → 統計數據製作圖表 → 版面設定 → 最後檢查」的固定步驟。
  • 建立標準操作順序：將常用操作整理成固定流程，減少臨場判斷時間。
  • 計時模擬訓練：用正式考試時間進行完整模擬，訓練時間分配能力。
  • 錯誤回溯分析：每次練習後記錄錯誤原因，再針對弱點重點補強，而不是重做整份題目。

學習收穫

透過乙級準備過程，我最大的收穫不只是軟體操作能力，而是：

流程拆解能力（把複雜任務變成步驟）
時間管理能力（在限制時間內完成高精度任務）
細節檢核能力（降低錯誤率）
問題分析能力（能找出錯誤來源，而不是只修正結果）
標準化思維（建立可重複使用的操作流程）

這些能力也讓我更理解資訊處理不只是「會用軟體」，而是如何透過邏輯與規劃，在有限時間內完成高品質成果。

TQC+ PowerPoint－專業級

學習內容

學習到簡報設計與資訊視覺化的基本與進階能力，包含投影片版面配置、文字與圖片編排、母片設定、動畫與轉場效果應用，以及整體簡報結構規劃。

遇到的困難

  • 版面配置不穩定：圖文比例不一致，導致畫面不夠整齊或重點不明確。
  • 操作熟練度不足：動畫、母片與格式設定不熟，容易花費過多時間在單一功能上。

解決方式

  • 建立固定製作流程：先設計架構（封面→內容→結論），再進行版面統一，最後才加入動畫與細節調整。
  • 拆解練習：不再一次做完整簡報，而是針對「單頁版面」與「功能操作」分段練習。
  • 錯誤回饋修正：每次練習後整理錯誤（如對齊問題、動畫過多），並調整下一次操作策略，而不是單純重做。

學習收穫

透過 TQC+ PowerPoint 的準備，我的收穫不只是簡報製作能力，而是：

視覺資訊整合能力（能將資訊轉化為清楚畫面）
版面邏輯規劃能力（提升結構與閱讀順序設計）
流程管理能力（建立穩定製作步驟）
錯誤修正能力（能分析問題而非只修正結果）

TQC+ Word－專業級

學習內容

學習到進階文書處理與文件排版能力，包含段落與字型格式設定、表格製作與調整、頁首頁尾設計、分欄排版、圖片與文字配置，以及長文件的整體結構規劃。

遇到的困難

1. 格式容易跑掉
段落間距、縮排與字型設定不一致，導致整體版面不整齊。

2. 操作步驟不熟，效率不足
功能分散（頁面設定、表格、圖片排版），初期常花時間找功能位置。

解決方式

面對這些問題，我調整學習方式，從「操作練習」轉為「流程與結構思維」：

1. 先規劃再製作
在開始排版前，先將文件拆解為「標題→內文→表格→圖片→頁尾」，建立整體結構。

2. 建立格式標準化習慣
統一設定字型、段落樣式與標題格式，減少重複調整時間。

3. 分段練習與功能拆解
不一次完成整份文件，而是針對表格、圖片排版等功能逐一練習。

學習收穫

透過 TQC+ Word 的準備，我的收穫不只是文書操作能力，而是建立了更完整的資訊處理能力：

文件結構規劃能力（先架構再內容）
格式標準化能力（提升一致性與效率）
問題分析能力（能找出錯誤來源）
細節控管能力（降低格式錯誤）

TQC+ Excel－專業級

學習內容

學習到試算表的基礎與進階應用能力，包含資料輸入與整理、公式與函數運用（如加總、平均、條件判斷）、資料排序與篩選、圖表製作，以及基本資料分析與報表呈現。

遇到的困難


1.公式與函數使用不熟
初期常混淆函數用途，例如 SUM、AVERAGE、IF 判斷條件設定錯誤。

2.資料邏輯容易出錯
資料排序或篩選後，容易影響原本計算結果，導致答案錯誤。

3.計算錯誤不易察覺
Excel 不會直接提示邏輯錯誤，常需反覆檢查才能發現問題來源。

解決方式

面對這些問題，我調整為「邏輯驗證＋結構化思考」的方式：

1.拆解公式邏輯
將函數拆成「條件 → 計算 → 結果」逐步理解，而不是直接套用。

2.建立檢查流程
每次完成後固定檢查三步驟：
公式是否正確
資料範圍是否正確
結果是否合理

3.分區處理資料
將資料分區整理（原始資料區／計算區／輸出區），避免混亂影響結果。

4.錯誤回溯分析
發現錯誤時不直接重做，而是追溯是「公式錯、範圍錯、還是邏輯錯」，再針對原因修正。

學習收穫

透過 TQC+ Excel 的準備，我不只是學會試算表操作，更建立了資料思維能力：

資料邏輯分析能力（理解資料如何影響結果）
函數應用能力（將問題轉化為公式）
錯誤診斷能力（找出問題來源而非只改結果）
流程規劃能力（資料分區與結構化處理）
數據整理能力（將資訊轉為可讀報表）

TQC+ 英文輸入－進階級

學習內容

學習英文鍵盤輸入的正確性與速度訓練，包含英文單字與句子的輸入方式、大小寫與標點符號的正確使用，以及提升盲打能力與鍵盤熟悉度。

遇到的困難

1. 速度與正確率無法兼顧
初期為了追求速度，容易出現拼字錯誤，導致整體成績下降。

2. 鍵盤不夠熟悉
英文單字位置不熟，常需要看鍵盤輸入，影響輸入效率。

解決方式

面對這些問題，我改變練習方式，從「追求速度」改為「建立穩定輸入系統」：

1. 先準確再加速
初期刻意降低速度，確保每個單字輸入正確，再逐步提升速度。

2. 固定指法與盲打訓練
訓練手指位置記憶，減少視覺依賴，提高輸入穩定性。

3. 計時模擬訓練
使用正式測驗時間反覆練習，建立時間壓力下的穩定輸入能力。

學習收穫

透過 TQC 英文輸入的訓練，我不只是提升打字速度，更建立了以下能力：

穩定輸入能力（速度與正確率平衡）
專注力與細節控制能力
時間壓力下的表現能力
持續訓練與自我改善能力

TQC+ Python 3 程式設計－進階級

學習內容

學習到基礎程式設計與邏輯運算能力，包含變數使用、輸入輸出、條件判斷（if/elif/else）、迴圈（for、while）、基本資料型態與簡單演算法應用。

遇到的困難


1.程式邏輯不熟，無法直接轉換問題
初期常看得懂題目，但無法拆解成程式步驟。

2.語法容易錯誤
常出現縮排錯誤、條件判斷寫錯或迴圈使用不正確。

3.除錯能力不足
程式出錯時，不知道問題是出在邏輯還是語法，導致修改效率低。

解決方式

面對這些問題，我調整學習方式，從「背語法」改為「建立程式邏輯思維」：

1.拆解問題流程
先將題目拆成「輸入 → 判斷 → 處理 → 輸出」四個步驟，再開始寫程式。

2.逐行測試程式
不一次寫完整程式，而是分段執行，確認每一步輸出是否正確。

3.錯誤分類與除錯
將錯誤分為「語法錯誤、邏輯錯誤、輸出錯誤」，並針對類型調整修正方式。

4.反覆練習題型規律
透過大量類似題目，整理常見解法模式（例如迴圈計算、條件篩選），建立解題模板。

學習收穫

透過 TQC+ Python 的學習，我不只是學會寫程式，而是建立了：

邏輯拆解能力（將問題轉換為步驟）
程式思維能力（用流程而非直覺思考）
除錯能力（能分析錯誤來源）
系統化解題能力（建立固定解題模式）
問題解決能力（從錯誤中修正並優化）

TQC+ App Inventor 程式設計

學習內容

學習到基礎行動應用程式設計能力，包含介面設計（UI設計）、元件配置、事件觸發機制、變數與邏輯控制，以及基本應用程式流程設計。

遇到的困難

1.邏輯連接不清楚
雖然是積木式程式，但仍需要理解「事件 → 條件 → 動作」的流程，初期容易接錯邏輯。

2.元件與功能搭配不熟
不清楚不同元件（按鈕、標籤、輸入框）如何正確搭配使用。

解決方式

先畫流程再寫程式
1.先將功能畫成流程圖（輸入→判斷→輸出），再轉換成積木程式。

功能拆解練習
2.不一次完成整個App，而是先練習單一功能（例如：按鈕變數、畫面切換），再逐步整合。

逐步測試（Debug思維）
3.每完成一小段功能就立即測試，確認問題出在哪一個模組。

錯誤分類修正
4.將問題分為「邏輯錯誤、元件設定錯誤、流程錯誤」，再分別調整，而不是整個重做。

學習收穫

透過 TQC+ App Inventor 的學習，我不只是學會做App，而是建立了：

流程設計能力（把功能轉成步驟）
模組化思維（將大問題拆成小功能）
邏輯判斷能力（事件與條件控制）
除錯能力（能定位錯誤來源）
專案整合能力（將多功能整合成完整應用）
